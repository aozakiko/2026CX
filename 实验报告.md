# STM32 创新项目实践实验报告

## 1. 整理说明

本次整理只关注当前工作区顶层名称以“实验”结尾的文件夹，压缩包、驱动包和图片目录不作为实验工程处理。

编号顺序依据各实验目录中 Keil 工程相关的用户界面/选项文件最后修改时间判断，主要参考 `*.uvguix.*` 与 `*.uvoptx`。这些文件通常会在 Keil uVision 打开、关闭或保存工程状态时更新；工程 XML 内部的 `DaveTm` 字段均为 0，因此未作为排序依据。

实验名称依据源码实际功能重新命名，时间只用于决定编号先后。

各实验内部主工程目录已统一命名为 `Project`，因此主 Keil 工程均位于 `*/Project/Template.uvprojx`。`04-内部温度传感器显示实验` 中另保留 `Reference-基础温度采集实验`，它是旧版仅串口/LED 输出的基础温度采集工程，不作为 04 号主工程。

经复核，原 `07-ESP8266-MQTT联网复现实验` 与 03 号实验高度重叠，且 03 号已包含更完整的 MQTT 调试、接收轮询和 LED 控制逻辑，因此删除原 07 号复现实验，并将后续实验顺次前移重新编号。

## 3. 实验内容概述

### 01 指针式电子时钟实验

工程在 `User/main.c` 中初始化 SysTick、串口、TFTLCD 和 RTC。程序通过 `RTC_Get()` 获取当前时分秒，同时使用圆形绘制、刻度线绘制和指针绘制函数在 TFTLCD 上实现指针式时钟，并同步显示数字时间。

主要模块包括 `APP/rtc`、`APP/tftlcd`、`APP/led`。

### 02 蜂鸣器控制实验

工程初始化 LED 与蜂鸣器 GPIO，在主循环中周期性翻转蜂鸣器输出状态，实现基础声光控制。该实验用于验证 GPIO 输出、延时函数和外设驱动封装。

主要模块包括 `APP/beep`、`APP/led`。

### 03 ESP8266-MQTT 联网调试实验

工程通过 `WiFi_Config()` 初始化 ESP8266 所在 USART3、CH/RST 控制引脚和中断接收缓存。主程序调用 `ESP8266_STA_TCP_Client_MQTT()`，完成 STA 模式联网、MQTT 用户配置、Broker 连接和主题订阅。该版本加入 `ESP8266_MQTT_Poll()`、订阅帧解析和 LED 控制处理，可持续轮询 `+MQTTSUBRECV` 数据，通过 MQTT 消息控制 LED，适合作为网络通信调试增强工程。

主要模块包括 `APP/esp8266`、`APP/led`。

### 04 内部温度传感器显示实验

工程使用 STM32 内部温度传感器通道进行 ADC 采样，并将温度值换算为摄氏度。增强版本在 `User/main.c` 中加入 TFTLCD 显示，将温度以 `+xx.xx C` / `-xx.xx C` 形式显示在屏幕上，同时保留串口输出。

主要模块包括 `APP/adc_temp`、`APP/tftlcd`、`APP/lcd`、`APP/led`。

### 05 FSMC-TFTLCD 显示实验

工程初始化 FSMC 接口驱动 TFTLCD，演示字符串显示、中文字库显示、色块填充、像素读取和图片显示。该实验验证了并口 LCD 的初始化时序、显存写入和基础图形接口。

主要模块包括 `APP/tftlcd`、`APP/led`。

### 06 光敏传感器采集实验

工程初始化光敏传感器采集模块 `lsens`，周期性读取光照强度值，并通过串口输出采样结果。该实验关注 ADC 输入、传感器数据读取和串口调试输出。

主要模块包括 `APP/lsens`、`APP/led`。

### 07 USB 虚拟串口通信实验

工程基于 USB Device CDC 实现 STM32 USB 虚拟串口。主程序初始化 USB 设备、TFTLCD 和 LED，在屏幕上显示 USB 连接状态，并在接收到 USB 虚拟串口数据后进行回发，实现基础 USB CDC 通信验证。

主要模块包括 USB Device、`APP/tftlcd`、`APP/led`。

### 08 温湿度光照环境监测实验

工程整合内部温度、DHT11 湿度和光敏传感器采集。`EnvMonitor_Read()` 汇总温度、湿度、光照百分比和估算 lux 值，主循环每秒输出人类可读日志与 `ENV,uptime_s,temp_x100,humidity,light_percent,lux` 格式的数据帧。

主要模块包括 `APP/env_monitor`、`APP/dht11`、`APP/esp8266`、`APP/led`。

### 09 ESP8266 温度 MQTT 发布实验

工程在 ESP8266/MQTT 联网基础上加入内部温度采样。程序订阅 `topic911` 用于 LED 控制，并周期性向 `stm32/temperature` 发布内部温度值。相较 03 号联网调试实验，该工程更接近传感器数据上云/局域网发布应用。

主要模块包括 `APP/adc_temp`、`APP/esp8266`、`APP/led`。

### 10 串口摄像头 TFTLCD 显示实验

工程将 PC 摄像头采集脚本与 STM32 显示端配合使用。PC 端脚本将摄像头画面转换为 128x64 RGB565 数据帧，经串口发送到 STM32；单片机端使用 `serial_image` 状态机校验帧头、长度、载荷和校验字节，收到完整帧后调用 `TFTLCD_DisplayColorFrame()` 显示到 TFTLCD。

主要模块包括 `APP/serial_image`、`APP/tftlcd`、`APP/oled`、`PC/send_camera_to_oled.py`。
